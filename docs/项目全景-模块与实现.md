# 项目全景：模块与实现

> 最近核对：v1.3.0（2026-07-24）。当前正式入口为 Tauri 桌面端与局域网 Web 端；旧 Qt 业务入口已移除。本文只记录当前实现。

## 一、总体架构

```text
core/              Python 纯业务逻辑，零 Qt 依赖，是唯一业务事实源
tauri-app/         React 19 + Vite 8 + Tauri 2 桌面端
web-app/           React 19 + Vite 8 局域网浏览器端
web_server.py      登录、注册审核、上传、任务、复核、下载与看板 API
web_control_gui.py Web 服务启停控制台，项目中唯一保留的 PySide6 界面
packaging/         Tauri Python sidecar 的 PyInstaller 入口和规格
tests/             unittest，使用合成 Excel 和临时目录
assets/            桌面端、Web 端和控制台共用品牌资源
```

数据链路：`React -> Rust/HTTP -> core.tauri_bridge -> core/*`。React 负责界面与状态，Rust 和 Web 服务负责白名单、进程、用户隔离与事件转发，不重复实现业务算法。

## 二、共享基建

### core/version.py

- 应用名称、`APP_ID`、发布者、更新地址和版本的单一事实来源。
- v1.3.0 使用 `VERSION="1.3.0"`、`VERSION_TUPLE=(1,3,0)`、`BUILD_DATE="2026-07-24"`。
- Tauri 发布同步 `tauri-app/package.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`；Web 同步自身 `package.json` 和锁文件。

### core/paths.py 与 core/settings.py

- `paths.resolve_output_dir()` 按 unified/beside/custom 模式创建中文功能目录，并用原子 `os.mkdir()` 抢占同分钟唯一目录。
- Web 任务通过 `FYT_WEB_OUTPUT_ROOT` 隔离到用户和任务目录。
- `settings.Settings` 校验固定字段、枚举、布尔值与范围；损坏配置备份后回落默认值，保存使用临时文件和 `os.replace()`。

### core/storage_lock.py

- `file_lock(path)` 为 JSON 索引提供跨进程文件锁。
- Windows 字节锁之外叠加进程内 `RLock`，防止同一进程多个线程互相越过锁。

### core/task_history.py

- SQLite 记录任务状态、耗时、输出目录、说明和 `request_id`。
- 自动迁移旧表并创建请求索引；取消使用单事务更新，完成回调只能更新 `running`，不会覆盖已取消状态。

### core/mapping_store.py、template_store.py、incremental_cache.py

- 字段映射按表头结构指纹复用人工确认结果。
- 模板中心管理模板族、版本、结构差异和显式迁移规则。
- 增量缓存按输入内容与参数生成键，命中前验证输出仍存在；读改写均加文件锁。
- Web sidecar 为每个用户设置独立 `FYT_INCREMENTAL_CACHE_PATH`，避免跨用户结果串用。

### core/common_core.py、header_detect.py、shape_detect.py

- 统一日期、时间、数值、名称和脏字符归一化；日期会验证真实日历值。
- `warn_if_uncached()` 检测 Excel 公式没有缓存值这一高风险输入。
- `load_data_only_stream()` 修复错误 dimension 后流式读取大工作簿。
- 表头识别先精确再包含，形态识别只作人工确认前的兜底。

### core/tauri_bridge.py

- 从 stdin 接受单次 JSON 请求，从 stdout 返回响应，从 stderr 发送带 `request_id` 的日志/进度事件。
- 所有动作走白名单；路径、必填列表和目录参数先校验，复杂返回值递归转换为 JSON 安全结构。
- 主动把标准流重设为 UTF-8，适配中文 Windows 和 PyInstaller sidecar。

### core/updater.py

- 清单和安装包地址必须为 HTTPS，新版本必须有合法 SHA-256。
- 下载写入 `.part`，长度和哈希通过后原子替换；网络、磁盘或校验异常会删除半包。
- 启动安装器前再次核验扩展名、哈希和伴随校验文件。

## 三、业务模块

### attendance_core.py

按姓名和完整日期键合并打卡来源，填入考勤目标表，计算班次、实际工时和异常报告。跨午夜仅对夜班规则生效，避免白班误加 24 小时。

### reconcile_core.py

把来源工时聚合到总表，再与劳务表逐日和总计对账；公式列优先读取缓存值，输出差异汇总与可信度报告。

### arrival_core.py

识别送货计划中的批次、供应商和剩余未收数量，按批次生成每日到料明细；批次记忆保存失败只记录日志，不破坏已生成文件。

### pivot_core.py

流式提取采购字段，统一规格和单位后聚合，并生成原生动态透视表与可信度报告。Web 缓存命中时把产物复制到当前任务目录，避免返回旧任务路径。

### purchase_core.py

按名称和规格分桶建立候选边，使用最小费用最大流，优先最大配对数、其次最大评分，避免贪心顺序漏配。

### delivery_core.py

合并物料清单和供应商明细，识别子表和字段后生成送货计划；所有表头列通过公共识别器定位，不写死列号。

## 四、工具模块

- `library.py`：自动分类、归档和索引；覆盖、删除、重新分类具备回滚边界和文件锁。
- `invoice_core.py`：递归扫描增值税专用发票 PDF，逐张复核后按月汇总。
- `currency_core.py`：金额转中文大写，拒绝 NaN 和 Infinity。
- `text_core.py`：去重、排序、清理、大小写、编号与信息提取；数字倒序时非数字稳定置底。
- `rename_core.py`：重命名预览、冲突检测、执行与撤销。
- `pdf_core.py`：合并、拆分、提取、删页，先读入内存避免 Windows 锁住源文件。
- `excel_tools_core.py`：合并、拆分、转换和纵向拼表；支持 CSV，按规范化表头重排列并拒绝结构缺失。
- `compare_core.py`：使用 Decimal 比较数字，保留前导零编码语义，重复键按最少差异配对。
- `preview_core.py`：只读前若干行和子表名称，为前端预览提供轻量数据。

## 五、Tauri 桌面端

- `tauri-app/src/` 包含 20 个导航页面、数据看板、业务表单、人工复核、任务中心、映射与模板中心、设置和更新。
- `FeatureUi.tsx` 统一文件/目录选择、拖放、运行、进度、取消、日志和结果。
- `useBridgeTask.ts` 订阅 Rust 转发事件并维护请求状态。
- `src-tauri/src/lib.rs` 只开放白名单命令，后台启动 sidecar，维护 `request_id -> pid` 以精确取消。
- release 只使用同目录 `FYTCoreBridge.exe`；debug 才回退项目 `.venv`。
- `scripts/build-tauri.ps1` 在构建前校验 Python、npm、Cargo 和 Tauri 四处版本一致。
- `scripts/visual-qa.mjs` 与 `functional-qa.mjs` 覆盖页面、响应式布局、动画降级和桥接状态。

## 六、局域网 Web 端

### web_server.py

- 标准库 `ThreadingHTTPServer` 托管 `web-app/dist`。
- SQLite 保存用户、会话、审核日志、上传和任务；密码使用随机盐与 PBKDF2-HMAC-SHA256。
- 注册账号必须由管理员审核；普通用户不能访问管理员接口。
- 上传和任务按用户 ID 隔离，静态和下载路径经过解析校验防目录穿越。
- 后台任务调用 `core.tauri_bridge`，转发进度与日志，支持取消、人工复核和结果下载。
- 看板按当前用户聚合七日趋势、状态分布、功能排行、最近任务和输出。

### web-app/

- 登录页不展示管理员凭据；注册、任务、看板、审核和复核均通过同源 `/api`。
- 响应式覆盖桌面与移动端，支持减少动画。
- `npm run build` 输出 `web-app/dist`，该目录不提交源码仓库。

### web_control_gui.py

- 项目唯一保留的 PySide6 界面，仅用于启动和停止 `web_server.py`。
- 使用 `pythonw.exe` 启动服务，命令窗口隐藏，日志仍由 `QProcess` 管道回传。
- `run-web.ps1` 默认后台隐藏启动并写 `web-data/logs`；`-Foreground` 用于诊断。

## 七、打包与测试

- `packaging/tauri_bridge_entry.py` 是 sidecar 极薄入口。
- `packaging/tauri_bridge.spec` 收集全部 `core` 子模块，显式排除 Qt、科学计算和开发依赖。
- `npm --prefix tauri-app run tauri:build` 先构建 sidecar，再生成 MSI 与 NSIS 安装包。
- Python 回归：`.venv\Scripts\python.exe -m unittest discover -s tests -p "test_*.py"`。
- Web 构建：`npm --prefix web-app run build`。
- Tauri 前端构建：`npm --prefix tauri-app run build`。
- Rust 回归：在 `tauri-app/src-tauri` 执行 `cargo test`。

## 八、维护约束

- `core` 不得导入 Qt、不得弹窗；错误向上抛，由前端翻译和展示。
- 新功能先实现 `core/<feature>_core.py` 和合成数据测试，再接入 Tauri/Web 白名单与页面。
- 输出目录统一走 `paths.resolve_output_dir()`；新功能必须注册 `FEATURE_DIRS`。
- 修改行为、签名或模块后同步本文；本文不记录已经删除的历史入口。
- 发布到 GitHub 时排除 `.venv`、`node_modules`、`dist`、`target`、`web-data`、sidecar 可执行文件和生成 schema。
